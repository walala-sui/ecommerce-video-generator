# Ecommerce Video Generator Skill

## 概述

此技能用于将电商人物图片转换为5秒的展示视频，保持人物姿态自然，动作小幅度。

## 使用流程

### 步骤1: 获取图片URL

首先使用图片上传技能获取图片的URL链接。

1. 将本地图片路径转换为WSL格式路径（如需要）
2. 使用图床上传图片获取URL
3. 如果tmpfiles.org的图片URL不被视频API支持，可尝试使用火山引擎对象存储或其他支持的图床

**注意：** 部分图床的图片URL可能不被视频生成API支持，如遇"image format not supported"错误，请尝试更换图床。

### 步骤2: 分析图片内容

使用MCP图片识别服务分析图片内容，获取详细的衣服特征描述。

调用 `mcp__MiniMaxcoding__understand_image` 工具：
- **prompt**: "详细描述这张图片中的衣服款式、颜色、图案、材质等特征"
- **image_source**: 图片路径（需转换为WSL格式）

根据识别结果，记录以下信息：
1. **衣服款式**：上衣、下装、连衣裙等
2. **颜色和图案**：主体颜色、图案类型
3. **材质**：面料特征
4. **人物姿态**：站姿、坐姿等
5. **手部状态**：是否手持物品
6. **面部朝向**：正面、侧面、角度

### 步骤3: 设计展示动作

根据图片中原有姿态设计动作，遵循以下原则：

**✅ 允许的动作：**
- 轻微的头部转动（不超过15度）
- 手臂轻微摆动（不离开画面）
- 身体轻微左右晃动（幅度小于5度）
- 面部表情自然变化
- 手指轻微动作（如果手上有物品）
- 衣角、头发等细节微动
- 服装颜色、图案、款式**严格保持与原图完全一致**
- 面料、光泽、纹理**严格保持与原图一致**

**❌ 禁止的动作：**
- 转身（背对镜头）
- 侧身展示（侧面对镜头）
- 手臂高举挥舞
- 行走、跑动
- 丢弃手中物品
- 大幅度动作导致画面与原图不符
- 大幅度快速的动作（如快速挥手、快速转身）
- 动作幅度过大或速度过快

**📦 手持物品处理：**
- 如果人物手持贵重物品（包、手机、珠宝等），动作应保持物品稳定
- 如果人物手持易碎物品（杯子、碗碟等），绝对不能丢弃或摇晃
- 轻微的手指活动可以保留，但物品位置保持不变

### 步骤4: 生成视频

使用豆包Seedance视频生成API：

```json
{
  "first_frame_image_url": "图片URL",
  "video_duration": 5,
  "resolution": "1080p",
  "fps": 24,
  "prompt": "描述保持原图的微动作展示"
}
```

**Prompt写作要点：**
- 强调"保持与原图一致"
- **强调服装款式、颜色、图案严格保持与原图完全一致**
- 使用"轻微"、"小幅"、"自然"、"缓慢"等词
- 具体描述允许的微动作
- 明确禁止转身、侧身、大动作、大幅度快速动作
- 强调动作缓慢、幅度小
- **Prompt长度控制在500字符以内**

**示例Prompt：**
```
奶牛纹V领宽松衬衫搭配白色高腰阔腿裤，白色背景，
服装款式、颜色、图案严格保持与原图完全一致，
仅轻微缓慢的面料波动，无转身动作，正面展示
```

### 步骤5: 下载视频

1. 从API响应中获取video_url
2. 将视频下载到原图片相同目录下
3. 文件命名为：原图片名 + "展示视频.mp4"

## 支持的图片路径格式

- **Windows**: `C:\Users\xxx\photo.jpg`
- **WSL**: `/mnt/c/Users/xxx/photo.jpg`
- **Linux/Mac**: `/home/xxx/photo.jpg`

## 输出要求

- **视频时长**: 5秒
- **分辨率**: 1080p
- **帧率**: 24fps
- **格式**: MP4
- **音频**: 无声音
- **文件保存位置**: 与原图片相同目录

## 错误处理

- 图片上传失败：尝试备选图床或提示用户
- 视频生成失败：调整prompt描述后重试
- 下载失败：提供视频URL供用户手动下载
